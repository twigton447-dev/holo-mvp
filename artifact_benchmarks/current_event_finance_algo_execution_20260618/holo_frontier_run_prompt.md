## Core Thesis
Execution alpha is becoming a control-plane problem, not only a signal problem. The winning system does not merely choose VWAP or TWAP; it continuously tests whether the current algorithm remains valid against regime, liquidity, peer behavior, venue economics, benchmark drift, portfolio weights, funding/settlement constraints, and compliance controls.


## HoloGov Technical Probe Requirement
Gov must not merely summarize prior turns. Every mission packet must include pointed technical probes the next model has to resolve. Gov should specifically challenge:

- order-flow assumptions and venue-routing logic
- benchmark gaming and benchmark conflicts
- peer/venue-relative comparisons
- portfolio-weight, active-risk, and funding constraints
- T+1 settlement, Treasury clearing, margin/collateral state
- Reg NMS tick/access-fee/odd-lot implications
- kill switches, pre-trade controls, audit logs, and human escalation
- AI-agent recency/extrapolation bias and unsupported source claims

The next model must answer the probes in the artifact or explicitly explain why a probe does not apply.

## HoloGov Validity and Selection Rule
Incomplete artifacts are automatic failures. Technical specificity cannot rescue a final report that ends mid-sentence, misses required sections, lacks required disclaimers, contains internal mission-packet residue, cites unknown source IDs, or falls outside the target word band.

Gov must maintain a repair ledger across turns:

- open issue -> repaired -> regressed -> still missing

Before final selection, Gov must apply deterministic gates to Turn 5 and Turn 6 before comparing quality. If Turn 6 is richer but incomplete or invalid, it must not be selected for clean scoring. Gov should select the complete validity-passing candidate or require a surgical repair/merge and keep the run diagnostic until repaired.

Gov should preserve winning finance features unless source grounding or validity gates reject them: intent provenance, funding formula, audit schema, tiered controls, and model-risk safeguards.
