## Status Key

| Status | Meaning |
|---|---|
| `benchmark_locked` | Frozen, adjudicated, hash-sealed. Do not modify. |
| `floor` | All architectures pass. No differentiation. Over-determined. |
| `diagnostic` | Construction issues, wrong catch mechanism, or unstable verdict. Not benchmark evidence. |
| `scout_only` | Holo-only or browser-only run. No full ablation. Solo behavior unknown or unconfirmed. |
| `retired` | Deprecated model dependency, design problem, or explicit retirement. Do not reopen. |

## Failure Shape Key

| Shape | Plain meaning |
|---|---|
| `nearestquestionerror` | Model answers the most proximate question (is this flag suspicious?) rather than the actual question (is every material condition satisfied?) |
| `frame_anchoring` | Model anchors on early document framing and cannot reweight later evidence |
| `procedural_obedience` | Model follows the workflow's clean signal without evaluating whether the workflow checked what matters |
| `authority_overread` | Model reads an authority figure's status notation as establishing a conclusion that figure did not evaluate |
| `scopesufficiencyerror` | Model resolves one necessary condition and treats it as sufficient, stopping before remaining conditions are verified |
| `noiseasassertion` | Model accepts an explanation or status note as establishing a fact it doesn't establish |
| `falsepositivebrittleness` | Model over-escalates on surface language that resembles a risk signal but isn't one in context |
| `selfcritiquereversal` | Self-critique architecture reverses a correct initial verdict |
| `consensuscollapse` | Homogeneous council collapses to wrong answer because all members share the same blind spot |
| `roleassignmentdependency` | Verdict shifts based on model identity (rotation-sensitive) rather than architecture |
| `council_induced_fabrication` | Ungoverned multi-agent council deliberation causes a model in a structured role (Evidence Checker, Skeptic) to invent primary document content — field values, section text, approval language — that does not exist in the sealed packet. The fabricated content is then used to justify a verdict. Distinct from hallucination in solo models: the council dynamic, not model confusion, generates the pressure that produces the invention. Canonical proof: MOD-RP-AOG-001-ABL-001, C-Gemini+Gemini-judge, 2026-06-05 — Evidence Checker invented J&A Section 6 containing a "forward-deployed rotatable pool" quantity justification absent from the packet. |
| `policy_vagueness_bypass` | Model treats conditional or permissive policy language ("may require," "as appropriate," "where applicable") as a negative permission — interpreting the absence of a mandatory directive as authorization to proceed. A conditional gate is unresolved until the condition is evaluated and the resolution is documented. An unresolved gate at the action boundary is a blocker regardless of policy phrasing. |
| `stale_clearance_laundering` | A prior clearance event, approval reference, or compliance note in item history or workflow records is treated as current authorization for a new transaction. The model borrows legitimacy from a past event and applies it to a present action that requires its own independent clearance. A reference is evidence that clearance was once sought — it is not a standing authorization for future automated execution. |
