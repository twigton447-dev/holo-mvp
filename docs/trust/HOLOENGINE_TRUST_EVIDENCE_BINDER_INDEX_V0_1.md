## Purpose And Claim Boundary

This binder index identifies HoloEngine trust evidence categories, existing draft source artifacts, missing evidence, likely owners, buyer-facing relevance, and claim risk.

This is an index for evidence collection. It is not a SOC 2 report, SOC 2 certification claim, SOC 2 readiness claim, HIPAA readiness claim, DPA/BAA coverage claim, provider zero-retention claim, or proof of production operating effectiveness.

All unverified controls are marked as `target_control_only`, `TBD`, or `missing`. Existing source artifacts are draft evidence only unless explicitly verified later.

Do not use this artifact to claim:

- SOC 2 readiness, compliance, certification, Type I completion, Type II completion, or auditor validation
- HIPAA compliance or HIPAA readiness
- DPA coverage, BAA coverage, subprocessor approval, or provider zero retention
- production control operating effectiveness
- verified encryption, retention, deletion, logging, support access, or tenant isolation
- that HoloEngine production customer payloads are never sent to providers or never stored
- that HoloVerify benchmark/evidence handling is equivalent to production customer data handling

## System Taxonomy And Mode Boundary

HoloEngine is the primary trust and assurance system taxonomy. HoloGov is in scope as part of HoloEngine runtime assurance. HoloVerify is a separate benchmark/evidence system and must not be treated as production customer payload handling. HoloChat and HoloBuild are connected surfaces only where they are deployed as part of a customer-facing production system.

| Surface | Binder Treatment |
| :--- | :--- |
| HoloEngine | Primary evidence binder subject. |
| HoloGov | Included as HoloEngine governance, selector, and control layer. |
| HoloVerify | Separate benchmark/evidence mode; supports evaluation discipline but not production privacy claims. |
| HoloChat | Connected surface where relevant; conditional production scope only. |
| HoloBuild | Connected surface where relevant; conditional production scope only. |
| HoloBrain | Conditional only if enabled in deployment. |

Production customer mode and HoloVerify benchmark/evidence mode must remain separate in storage, access control, retention policy, audit logging, and claim language. Production customer mode is target-designed for bounded ALLOW/ESCALATE review with metadata-only audit logging by default. Benchmark/evidence mode may preserve frozen prompts, raw provider outputs, scoring maps, trace files, runtime manifests, failure autopsies, provenance audits, and operational notes for benchmark integrity.
