Opening comment
The problem: AI agents can now execute payments, create users, delete files. Most implementations ask one model once and execute on the result. A sufficiently crafted context can get any single model to rationalize nearly anything.

Holo is a verification API that runs an adversarial loop before the action executes:

1. Three models (OpenAI, Anthropic, Google) evaluate independently — each in a different adversarial persona (Initial Assessment → Assumption Attacker → Edge Case Hunter, etc.)
2. Each model gets the full prior turn history and is structurally required to challenge it
3. A deterministic governor (not a model) makes the final call based on coverage matrix + convergence detection
4. Returns { decision: "ALLOW" | "ESCALATE", turn_history, coverage_matrix, elapsed_ms }

Convergence detection (delta=0 for 2 consecutive turns after min 3 turns) exits early on easy cases. Most evaluations complete in 3–5 turns. Hard cases go to 10.

We tested it on a BEC scenario. Every solo frontier model returned ALLOW. The adversarial council returned ESCALATE on turn 3 — the combination of domain age + routing change + urgency language only became HIGH-confidence through cross-model pressure.

API is live. 3 free evaluations at holo.ai. Curious what use cases you'd want to test.
